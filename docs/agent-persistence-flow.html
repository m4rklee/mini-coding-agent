结果整理
tool 和 retry 会回到“调用未超限?”重新组 prompt；final 或超限会进入结果整理并返回。每一次模型调用都会保存一个完整 prompt 文件。

初始化设置

创建或加载 session

  • id, created_at, workspace_root
  • history, memory, checkpoints
  • runtime_identity, resume_state
  • 文件：.mca/sessions/<session_id>.json

规范化 memory

  • working.task_summary, working.recent_files
  • episodic_notes, file_summaries
  • task, files, notes 为兼容镜像
  • next_note_index, durable_topics

评估恢复状态

  • 读取 checkpoints.current_id
  • 比较 schema_version, key_files.freshness
  • 比较 runtime_identity
  • 更新 resume_state.status, stale_paths

初始化落盘时间

  • 新建或恢复 mca 实例时保存 session
  • 恢复 session 时也会重算 resume_state
  • 过期文件摘要可能在此时被清理

用户发送消息 / 任务建构

更新当前任务摘要

  • memory.working.task_summary = user_message
  • memory.task 同步为兼容字段

追加用户历史

  • history[] 新增 { role: "user", content, created_at }
  • record() 立即保存 session

创建 TaskState / run 目录

  • run_id, task_id, user_request
  • status = running, attempts = 0, tool_steps = 0
  • 文件：.mca/runs/<run_id>/task_state.json

trace: run_started

  • event = run_started
  • task_id, user_request, created_at
  • 文件：.mca/runs/<run_id>/trace.jsonl

每轮循环：上下文建构 / 调用模型

更新 task_state

  • 每轮开始：attempts += 1
  • 写入 task_state.json
  • 超限条件：tool_steps < max_steps 且 attempts < max_attempts

构建 prompt 时可能更新 session

  • 刷新 workspace_fingerprint, prefix_hash
  • 重算 resume_state
  • 清理过期 file_summaries
  • 更新 runtime_identity

保存完整 prompt

  • 每次模型调用前保存一次
  • 文件：prompts/attempt_001.txt
  • 第二轮为 attempt_002.txt，依次递增

trace: prompt/model

  • prompt_built: prompt_metadata, prompt_path
  • model_requested: attempts, tool_steps, prompt_cache_key
  • model_parsed: kind, completion_metadata

工具调用分支

更新 TaskState

  • tool_steps += 1
  • last_tool = name
  • 写入 task_state.json

更新 memory

  • read_file: recent file + file summary + episodic note
  • write_file/patch_file: recent file + 清理旧 file summary
  • 失败/拒绝/partial success 可能追加 process note

追加工具历史

  • history[] 新增 { role: "tool", name, args, content, created_at }
  • 工具成功、失败、拒绝都会作为文本结果写入 history

trace: tool_executed

  • name, args, result, duration_ms
  • tool_status, tool_error_code, risk_level
  • affected_paths, workspace_changed, diff_summary

Checkpoint 创建

写入 session.checkpoints

  • 触发时机：tool_executed, run_finished, step_limit_reached, retry_limit_reached, freshness_mismatch, workspace_mismatch, context_reduction
  • 字段：checkpoint_id, parent_checkpoint_id, schema_version, created_at
  • 任务现场：current_goal, completed, excluded, current_blocker, next_step
  • 文件现场：key_files, freshness
  • 运行身份：runtime_identity
  • 同时更新：checkpoints.current_id, task_state.checkpoint_id

trace: checkpoint_created

  • checkpoint_id
  • trigger
  • created_at

落盘位置

  • checkpoint 本体保存在 session 文件里
  • 不是单独的 run 文件
  • run 里只通过 trace/report/task_state 引用 checkpoint id

retry / final / 超限结果整理

retry 分支

  • history[] 追加 assistant runtime notice
  • 保存 session
  • 写 task_state.json
  • 不创建 checkpoint，回到下一轮

final 成功整理

  • history[] 追加 assistant final
  • status = completed
  • stop_reason = final_answer_returned
  • final_answer = final

超限停止整理

  • 模型请求超限：retry_limit_reached
  • 工具步数超限：step_limit_reached
  • status = stopped
  • final_answer 为停止说明

长期记忆

  • 只有用户有“记住/保存/记录”等意图时触发
  • 写入 .mca/memory/MEMORY.md
  • 写入 .mca/memory/topics/*.md
  • report 记录 promotions/rejections/superseded

trace: run_finished

  • status
  • stop_reason
  • final_answer
  • run_duration_ms

report.json

  • run_id, task_id, status, stop_reason
  • tool_steps, attempts, checkpoint_id
  • task_state, prompt_metadata, redacted_env

一眼看懂：三条落盘线

  1. session 线：保存长期会话状态，包含 history、memory、checkpoints、runtime_identity、resume_state。
  2. run 线：保存单次 ask 的审计工件，包含 task_state、trace、prompt、report。
  3. durable memory 线：只有明确长期记忆意图时才写入 `.mca/memory`。
最容易混淆的是 checkpoint：它不是 run 目录里的单独文件，而是 session 内部的恢复点。run 目录记录“这次怎么跑”，session 记录“会话现在是什么状态”。
